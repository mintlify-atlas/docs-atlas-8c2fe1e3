## Bienvenido al sistema

El **Sistema de Distribución de Suministros** de Farmcorp permite a los Puntos de Venta (PDV) y departamentos internos solicitar suministros de forma digital, con validación de cupos, ventanas de tiempo configurables por zona y un flujo de aprobación centralizado.

```
PDV realiza pedido → Sistema valida cupo y ventana → Notificación email → Aprobador revisa → Aprobado / Rechazado
```

## Flujo del sistema

<Steps>
  <Step title="Inicio de sesión">
    El usuario se autentica con credenciales corporativas. El sistema detecta automáticamente su rol, departamento y PDV asignado.
  </Step>
  <Step title="Selección de suministros">
    El usuario selecciona el tipo de suministro, busca artículos y los agrega al carrito. El sistema valida cupos en tiempo real.
  </Step>
  <Step title="Envío del pedido">
    Al confirmar, el servidor recalcula todos los precios desde la base de datos y valida la ventana de pedidos para la zona.
  </Step>
  <Step title="Notificación automática">
    Se genera un CSV con el detalle del pedido y se envía por email al equipo responsable.
  </Step>
  <Step title="Aprobación">
    El aprobador revisa el pedido, puede editar cantidades y finalmente aprueba o rechaza con un motivo obligatorio.
  </Step>
</Steps>

## Características principales

<CardGroup cols={2}>
  <Card title="Control de cupos" icon="shield-check">
    Cada PDV tiene un cupo asignado por grupo. El sistema bloquea pedidos que excedan el límite disponible.
  </Card>
  <Card title="Ventanas de pedido" icon="calendar">
    Los pedidos solo pueden realizarse en los días configurados por zona comercial o departamento.
  </Card>
  <Card title="Autenticación LDAP" icon="key">
    Integración con Active Directory de Farmcorp para autenticación corporativa en producción.
  </Card>
  <Card title="Reportes Excel" icon="table">
    Exportación de reportes detallados con filtros por período, PDV, estado y tipo de suministro.
  </Card>
  <Card title="Precios desde BD" icon="lock">
    Los precios siempre se recalculan en el servidor. Los valores del frontend nunca se aceptan.
  </Card>
  <Card title="Panel administrativo" icon="settings">
    Gestión completa de usuarios, suministros, PDVs, proveedores y configuración del sistema.
  </Card>
</CardGroup>
